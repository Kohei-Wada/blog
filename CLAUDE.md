## Project Overview

This is a personal blog built with Astro, a modern static site generator. The site is deployed to Netlify at <https://wada-dev.com/>. It features a bilingual blog (Japanese/English) with tag-based categorization and RSS feed support.

## Development Commands

### Core Development

- `npm run dev` - Start development server at localhost:4321
- `npm run build` - Build for production to ./dist/
- `npm run preview` - Preview production build locally

### Testing

- `npm run test` - Run tests interactively with Vitest UI
- `npm run test:ui` - Open Vitest UI in browser for visual test management
- `npm run test:run` - Run all tests once in CI mode
- `npm run test:coverage` - Generate test coverage report
- `vitest run tests/unit/consts.test.ts` - Run specific test file
- `vitest run -t "should format date"` - Run tests matching pattern

### Code Quality

- `npm run lint` - Run ESLint on all .js/.ts/.Astro files
- `npm run lint:fix` - Run ESLint with auto-fix
- `npm run typecheck` - Run Astro's TypeScript type checking
- `npm run markdown:lint` - Lint markdown files
- `npm run markdown:fix` - Fix markdown lint issues
- `npx prettier --write .` - Format all code with Prettier

### Content Management

- `npm run check:posts` - Verify every slug has both a ja and en post (parity)
- New posts are created by hand as a ja + en pair under `src/content/blog/{en,ja}/`; see the `write-post` skill

### Security & Maintenance

- `npm audit` - Check for security vulnerabilities
- `npm audit fix` - Auto-fix security issues

## Architecture

### High-Level Overview

This is a static site generated by Astro with MDX support. The build process transforms Markdown/MDX content into static HTML pages with optimized assets. Key architectural decisions:

- **Content Collections API**: Type-safe content management with Zod schema validation
- **File-based Routing**: Pages in `src/pages/` automatically become routes
- **Component Islands**: Interactive components only hydrate where needed
- **Static Generation**: All pages pre-rendered at build time for optimal performance

### Content Management System

- **Blog Posts**: `src/content/blog/` - Markdown/MDX files with frontmatter
- **Schema Definition**: `src/content.config.ts` - Zod schema for frontmatter validation
- **Content Types**: `src/types/index.ts` - TypeScript interfaces for content structures
- **Template**: `src/content/blog-template.md` - Starter template for new posts
- **Parity Check**: `scripts/check-post-parity.mjs` - Enforces a ja + en pair per slug

### Page Routing Strategy

Both locales are served under a prefix (`/en/...`, `/ja/...`); `/` redirects to
`/en`. The logic pages live once under `[lang]/` and `getStaticPaths` emits both
locales (filter / hrefs / i18n derived from `Astro.params.lang`). Prose pages stay
two files (`en/`, `ja/`). Internal links must be locale-prefixed вЂ” a bare
`/blog/<slug>` no longer exists.

### Styling Architecture

- **Global Styles**: `src/styles/global.CSS` - Base styles and CSS variables
- **Component Styles**: Scoped `<style>` blocks in Astro components
- **Typography**: Atkinson font family from `public/fonts/`
- **Responsive Design**: Mobile-first approach with breakpoints at 768px
- **Dark Mode Support**: CSS variables for theme switching

### Build & Deployment Pipeline

1. **Content Processing**: Markdown в†’ AST в†’ HTML with syntax highlighting
2. **Asset Optimization**: Images processed with Sharp, fonts preloaded
3. **Static Generation**: All routes pre-rendered to HTML
4. **Sitemap Generation**: Automatic sitemap.xml creation
5. **Deployment**: Netlify CI/CD on push to main branch
6. **Daily Build**: Scheduled build at 00:00 JST via GitHub Actions

### Optional Frontmatter

```yaml

```
