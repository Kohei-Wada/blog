こんにちは、Koheiです。

東京でプログラマーとして働いており、普段はリモートワーク中心です。
仕事用マシンはWindowsですが、実際の開発作業はほとんどWSL上のArch Linuxで行っています。
一方で、個人環境は完全にArch Linuxをベアメタルでインストールしており、Windowsは削除済みです。
軽量で自分の手で構成をコントロールできる環境が好きです。

もともと数学が好きで、関数をグラフ化するためにPythonを使い始めたのが、プログラミングの出発点でした。
抽象的な考え方や構造に惹かれて、今でもHaskellのような関数型言語に関心があります。

LaravelやPHPは業務で扱う場面もありますが、特に好んで使っているわけではありません。
また、VSCodeやCursorといった重いGUI IDEよりも、自分で整えたターミナル中心の環境が肌に合っています。

フロントエンドにはこれまで積極的に取り組んできませんでしたが、「ブログを自作すればきっと向き合える」と思い、Astroを選んでこのブログを作りました。
自分の開発記録や思考ログが、未来の自分やどこかの誰かの役に立てばと思っています。

## 好きなもの・よく使うツール

- Arch Linux（WSLでもベアメタルでも運用中）
- Neovim（昔vimを使っていましたが、neovimのほうが軽い感じがします）
- Bash / ShellScript（自動処理・cronなど）
- Python（グラフ描画・スクリプト用途）
- Hyprland（Wayland時代の軽量WM）
- Haskell（関数型言語）

## あまり好きではないもの

- Laravel / PHP（使ってはいるけど積極的に選ばない）
- VSCode / Cursor（GUI IDEは重くて好みではない）
- Windows（個人環境からは完全に撤去）

## 関心のあること

- Rust（CLIツールや自動処理の開発）
- Zettelkastenによる知識の整理と成長
- 音声認識の非同期処理、最小構成システム設計

ソースコードやdotfilesは GitHub に公開しています。興味があればぜひ覗いてみてください：
[GitHub.com/Kohei-Wada](https://github.com/Kohei-Wada)

## このブログについて

このブログは、普段の開発や検証のメモをアウトプットする場として作りました。
技術的な整理をするだけでなく、誰かに見てもらってフィードバックをもらうことを目的としています。

フロントエンドには苦手意識がありますが、あえてブログを自作することで、
自分にプレッシャーをかけて「触らざるを得ない」環境を作りました。

内容は日々の作業ログや技術検証、思考の断片などが中心です。
誤りや気づき、アドバイスなどがあれば、気軽にGitHubやSNSなどでフィードバックをもらえると嬉しいです。

## エディターの遍歴

- Atom（最初に使ったエディター）
- Atom + vim モード（キーバインドの快適さに目覚める）
- VSCode + vim 拡張（ちょっと重さが気になり始める）
- Vim（VimバインドにするくらいならVimを使ったほうがいいと思った）
- Neovim（今のメイン。Lua + プラグイン地獄に片足突っ込む）

## その他の活動

GitHubでは自作ツールの公開のほかに、オープンソースプロジェクトへの貢献も行っています。
